Version Control Systems (VCS)

Using a VCS is a crucial tool for a modern developer

What is a Version Control System?

In modern web development, developers often have to juggle multiple different projects often collaboratively with other developers or designers. Using a VCS allows multiple developers to work on the same projects without the risk of conflicts and lets us keep several versions of our code which we can revert back and forth. It is also handy to have multiple versions of the same product to allow us to make changes and test things out without affecting the source code of a product which may already be live.

Version control systems come in a variety of shapes and sizes but are usually either Local or Remote:

Git (Local)

  • Saved locally on your machine ✓
  • Takes snapshots of your code whenever you make a commit ✓

GitHub (Remote)

  • Saved to a remote machine ✓
